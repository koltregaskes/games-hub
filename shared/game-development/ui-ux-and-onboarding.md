# UI, UX, and Onboarding

## What good looks like

- The first ten minutes are understandable without reading code or a design doc.
- Menus, HUD, and onboarding feel like part of the game, not excuses around it.
- Players can recover from confusion quickly.
- Accessibility and readability are treated as product quality.

## Minimum studio-standard version

- A readable title flow.
- A first-run path.
- A readable HUD or information layout.
- Settings for the essentials that matter now.

## Common mistakes

- Over-explaining the game before letting the player touch it.
- Hiding important state in tiny UI elements.
- Building stylish menus that delay the start too much.
- Forgetting mobile or controller readability if those inputs are in scope.

## Practical checklist

- Define the first screen and first action.
- Decide how the player learns the core rule.
- Check HUD readability during motion, stress, or combat.
- Add the settings players actually need now.
- Review the flow with someone who did not build it.
