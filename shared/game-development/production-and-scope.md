# Production and Scope

## What good looks like

- The game has one clear vertical slice that already sells the fantasy.
- The team can explain the next review gate without opening the repo.
- Scope is being cut deliberately, not accidentally.
- A reviewer can play the slice and understand success, failure, and replay value.

## Minimum studio-standard version

- A title or menu flow.
- One complete loop.
- One readable end state.
- A clear list of what is intentionally not included yet.
- A review route or stable build entry point.

## Common mistakes

- Building menus, lore, and systems before the loop feels good.
- Adding a second zone or second mode before the first one is replayable.
- Confusing "more content" with "more proof."
- Letting every idea become a milestone.

## Practical checklist

- Write the current gate in one sentence.
- Name the one feature the build is trying to prove.
- Cut anything that does not help that proof.
- Decide what evidence will count as signoff.
- Record the next trigger for desktop migration, if any.
