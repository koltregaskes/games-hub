# Agents, Handoffs, and Studio Workflow

## What good looks like

- Agents get bounded tasks, not vague ambition.
- Handoffs are readable by a non-coder.
- The manager can see what changed without reconstructing the whole thread.
- Public-safe knowledge and private ops notes stay clearly separated.

## Minimum studio-standard version

- One handoff per game or domain.
- One intake format for verification or research.
- One place where current standards live.
- One rule for what must stay private.

## Common mistakes

- Letting every session invent its own structure.
- Asking an agent to "finish the whole game" without a gate.
- Mixing operational notes into public repos.
- Accepting agent output without evidence or review criteria.

## Practical checklist

- Write the current target in one paragraph.
- Define the editable surface.
- Define what evidence is required.
- Define who curates the final standard.
- Keep shared knowledge in one reusable layer.
