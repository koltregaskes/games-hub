# Graphics

## What good looks like

- The game has a visual lane a screenshot can communicate instantly.
- Assets, lighting, and UI feel like they belong to the same world.
- The art direction is specific enough to guide future work.
- Performance and readability are treated as part of the visual design.

## Minimum studio-standard version

- One visual identity sentence.
- A controlled palette.
- One environment mood.
- One rule for how UI sits on top of the playfield.
- A simple asset sourcing rule.

## Common mistakes

- Pulling assets from five styles and hoping shaders fix it.
- Over-detailing backgrounds before the core interaction reads well.
- Letting UI fight the scene for attention.
- Mistaking technical fidelity for visual identity.

## Practical checklist

- Define palette, mood, and camera feel.
- Define what "placeholder" means visually.
- Choose whether style comes from lighting, texture, shape language, or all three.
- Test one screenshot at thumbnail size.
- Confirm that gameplay feedback still reads inside the chosen art style.
