# How To Write A Game Review

## Start with the evidence

Before you write a single line, lock down the context:

- build version
- platform
- input method
- hours played
- whether the main loop was completed
- whether any live-service or online features were unavailable
- whether the build was public, private, demo-only, or review code

If the evidence is thin, do not pretend otherwise. Change the format to a preview or a review in progress.

## Use a structure that helps the reader fast

### 1. Hook

Start with the clearest truth about the experience.

- What feeling does the game create?
- What tension or fantasy defines it?
- What is the one sentence a reader should remember?

### 2. What the game is

Explain the core shape quickly.

- genre
- fantasy
- loop
- current stage if it matters

Do not spend half the review reciting features.

### 3. What works

Pick the strengths that actually drive the experience.

- the loop
- the atmosphere
- the controls
- the pacing
- the mission structure
- the strategic pressure

Explain why each strength matters in play.

### 4. What does not work

This is where the review earns trust.

- weak onboarding
- thin content
- poor closure
- unstable performance
- repetitive structure
- confused UX

Be specific. Readers should be able to imagine the problem.

### 5. Technical and usability reality

Even strong games lose trust if they waste the player’s time.

Call out:

- crashes
- bugs
- frame pacing
- save issues
- readability problems
- controller or keyboard friction
- settings and accessibility gaps

### 6. Who this is for

A good review is partly a matchmaking service.

Say:

- who will love this
- who should wait
- who should skip it

### 7. Verdict

End with a short, confident conclusion.

The verdict should feel inevitable from the body of the review.

## Questions that improve any review

- What is the game actually asking the player to do, minute to minute?
- Does the game get better, flatter, or worse over time?
- Is the game readable to a first-time player?
- Does the ending or stopping point feel earned?
- Is the game ambitious in a way that matters, or just noisy?
- Would I recommend this today, not after three hypothetical patches?

## Common review mistakes

- confusing premise with execution
- praising intention instead of shipped reality
- turning the review into a plot summary
- listing pros and cons without building an argument
- using bugs as the whole review instead of part of it
- giving a score that does not match the written case

The best reviews feel like guided judgment, not admin.
